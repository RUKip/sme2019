Attempt to fix bug #2817321: Exception on startup.
This bug was caused by a back-incompatibility introduced by BufferSets. Previously, the perspective.xml file specified the split configuration by means of buffers (there was a single global buffer-set, shared by all edit panes). BufferSets changed the format of the split configuration to include a specification of buffer-sets, and the split is now specified by means of buffer-sets.

shlomy 2009-07-10 09:40 #17 Rev.: 15681

Hopefully fixed the restoring of views from the saved perspective for dual-monitor environments. A previous bug fix I made was meant to solve the case where a user moves from a dual-monitor setup to a single monitor setup, with the same settings directory, in which case if a view was previously on the second monitor, its startup location will be outside the single monitor, thus it won't be on the screen.
My previous fix used GraphicsEnvironment.getMaximumScreenBounds, which only returns the bounds of the current display. For dual-monitor setups, this is not sufficient, need to iterate all the devices to find the bounds.
