Vampire0 2007-06-28 17:33 #34 Rev.: 9871

Glob translation to regexes got a new feature. If you start a glob now with the sequence "(re)" this prefix is discarded and the remaining string is interpreted unchanged as regex. If you need a glob starting with "(re)" you have to escape it and write "\(re)" instead. So now you can use EVERYWHERE a glob is expected a normal regex instead and use their full power.
