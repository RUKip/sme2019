kerik-sf 2013-02-20 19:20 #13 Rev.: 22807

Insert_Buffer_Properties.bsh is UTF-8 - bug #3311234

Macros can't be assumed to be all UTF-8.
They have always been read using platform's default encoding
so it's not changed to not break user's macros.

The only core macro with non ascii is Insert_Buffer_Properties.bsh
I add a buffer-local-property encoding=UTF-8 to remove a warning
on windows, where it's not UTF-8 by default, but it works only
for people having the autodetector active.
