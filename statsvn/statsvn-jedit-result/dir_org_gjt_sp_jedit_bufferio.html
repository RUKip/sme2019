Changed the behavior when a file could not be loaded due to strict error
about encoding. The file is now loaded permissively. This makes it
possible to fix the error by editing and saving, and to guess what type
the file actually is.

This change requirs adding a method to the interface Encoding, assuming
no one, but jEdit core itself, implements it for now.

http://www.nabble.com/Re%3A-jEdit---file-encodings-td16482966.html
