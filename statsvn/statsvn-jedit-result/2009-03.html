shlomy 2009-03-20 15:08 #14 Rev.: 14805

(Hopefully) Fixed bug #2671628: perspective.xml not saved on exit
The startupDone vector is used to find out when all views have been laid out. The DockableLayoutSetter class is used to defer layout of all views loaded from perspective.xml to the AWT thread. Each instance lays out a different view, and marks in the startupDone vector when done. This was done in order to eliminate false dependencies between the various setters, but the index provided to each layout setter was based on the view count, which, in rare situations may be incorrect.
Changed the index to use the number of items in the startupDone vector, protecting it with "synchronized" to ensure no two instances get the same index.
